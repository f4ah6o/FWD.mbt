## 作業ログ

- `src/workbench_v6/` を追加した。
  - guided form builder の state を定義した。
  - builder から `@schema.Schema` を直接構築する変換を追加した。
  - `resolve -> validate -> emit -> runtime preview` を 1 つの pure evaluation にまとめた。
- `src/ui/workbench_v6/` を追加した。
  - plain HTML と MX HTML を 1 つの renderer から出す構成にした。
  - `ViewContext` を使い、root と form の MX wiring を切り替えるようにした。
  - validation section と preview section を deterministic に描画するようにした。
- `src/api_v6/` を追加した。
  - `GET /v6/workbench`
  - `GET /v6/workbench?format=mx`
  - `POST /v6/workbench/preview`
  - `POST /v6/workbench/preview?format=mx`
  - の pure routing を実装した。
- form body は最小の `application/x-www-form-urlencoded` 解析を自前で追加した。
  - これにより guided form builder を JSON body 前提にせず扱えるようにした。
- fixture と test を追加した。
  - logic test
  - ui fixture test
  - api fixture test
- `V6_0_PLAN.md` を、旧来の view-only 説明から workbench first slice の説明へ更新した。
- `README.mbt.md` に v6 workbench surface を追記した。

## 学び・メモ

- `@schema.Schema` を canonical internal model に置くと、YAML を編集 UX の中心にしなくても compiler / runtime を自然に再利用できる。
- first slice では browser transport を大きく作り込まなくても、固定 slot の guided form と deterministic HTML fixture で十分に価値を出せる。
- `sort_by` の順序感が直感と逆に見える箇所があり、preview fixture は実出力に合わせて固定する必要があった。
- tmpx の text node は `&` や `->` を HTML escape するので、fixture も escape 後の文字列で固定する必要がある。
